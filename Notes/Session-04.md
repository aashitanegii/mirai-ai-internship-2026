# 📑 Streamlit Sidebar

Instead of placing every widget on the main page, Streamlit provides a **Sidebar** to organize controls and settings.

Example:

```python
st.sidebar.title("Settings")
```

or

```python
with st.sidebar:
    st.title("Settings")
```

### Common Uses

* Model Selection
* Image Settings
* Theme Options
* Navigation
* AI Controls

Using a sidebar makes the application cleaner and easier to use.

---

# 🎚️ Intensity Slider

A slider allows users to select values from a range.

Example:

```python
intensity = st.slider(
    "Creativity",
    0,
    100,
    50
)
```

Possible Uses

* Creativity Level
* Image Style Strength
* AI Temperature
* Detail Level

Instead of fixed values, users can customize the AI output.

# 💬 Maintaining Chat History

Normally, every interaction refreshes the Streamlit app.

To remember previous conversations, Streamlit provides:

```python
st.session_state
```

Example:

```python
if "messages" not in st.session_state:
    st.session_state.messages = []
```

Benefits

* Stores previous chats
* Maintains conversation flow
* Makes the application behave like ChatGPT

# 🤝 Hugging Phase

The instructor explained the **Hugging Phase** as the stage where our application starts communicating with an AI model.

Instead of only creating the interface, our application now becomes AI-powered.

Workflow

```text
User Input
      ↓
Application
      ↓
AI Model
      ↓
Generated Output
```

This is the phase where AI processing actually happens.

---

# 🖼️ AI Image Generation

Instead of generating text, AI models can generate images from natural language prompts.

Example Prompt

> "A futuristic football stadium under the northern lights, cinematic lighting, ultra realistic."

Workflow

```text
User Prompt
      ↓
Image Generation API
      ↓
AI Model
      ↓
Generated Image
      ↓
Display in Streamlit
```

---

# ✍️ Prompt Engineering for Images

Image generation models understand descriptive prompts.

A good prompt should include:

* Subject
* Style
* Colors
* Lighting
* Background
* Camera Angle
* Quality

Example

```text
A futuristic football stadium,
sunset,
cinematic lighting,
ultra realistic,
4K,
highly detailed.
```

The more descriptive the prompt, the better the generated image.

# 📤 Output (O/P)

The **Output** is the final result generated by the AI after processing the user's input.

Examples

Input

> "Generate a football stadium."

Output

* AI-generated image

Input

> "Explain Machine Learning."

Output

* AI-generated text

Different AI models produce different types of outputs.

# 📷 Binary Stream

Image generation models do not return normal text.

They return the generated image as a **Binary Stream (Bytes)**.

The application converts this binary data into an image that users can view.

Workflow

```text
Prompt
      ↓
AI Model
      ↓
Binary Stream (Bytes)
      ↓
Application
      ↓
Displayed Image
```

# 🌐 REST API

**REST API (Representational State Transfer)** is a standard way applications communicate with AI services over the internet.

It follows the **Request → Response** model.

Workflow

```text
Application
      ↓
REST API
      ↓
AI Server
      ↓
Response
```

Examples

* Gemini API
* OpenAI API
* Pollinations AI
* Weather API
* GitHub API

---

# 🔄 HTTP Methods

## GET

Used to **retrieve** data from a server.

Example

```text
GET /images
```

Purpose

* Read data
* Fetch information
* Retrieve AI resources

---

## POST

Used to **send** data to a server.

Example

```text
POST /generate-image
```

Purpose

* Send prompts
* Upload files
* Generate AI responses
* Create new resources

---

# 📤 API Request Structure

Every API request generally contains:

* Endpoint (URL)
* HTTP Method
* Headers
* Body (Optional)

Example

```text
POST /generate-image

Headers
Authorization: API_KEY

Body
{
   "prompt": "A futuristic football stadium"
}
```

---

# 📥 API Response Structure

After processing the request, the server sends back a response.

Example

```json
{
   "status": "success",
   "image_url": "https://..."
}
```

The application reads this response and displays the generated result.
